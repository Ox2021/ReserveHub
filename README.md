# 机房预约系统

## 1.1 项目简介

* 学校现有几个规格不同的机房，由于使用时经常出现"撞车"现象,现开发一套机房预约系统，解决这一问题。

# 代码提交规范
* feat: 新增功能（feature）
* fix: 修复 Bug
* docs: 修改文档（如 README 文件等）
* style: 代码样式的更改（如空格、格式化等，不影响逻辑）
* refactor: 代码重构（既不修复 Bug 也不添加功能）
* perf: 性能优化
* test: 增加/修改测试代码
* chore: 其他杂项修改（如依赖更新、构建工具配置等）
